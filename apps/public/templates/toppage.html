WAWプラグイン仕様

概要

WAWはJavascriptで書かれた音源のプラグインシステムです。対応ブラウザはWebAudioAPI/AudioDataAPIに対応しているブラウザ(Chrome、Safari、Firefoxなど)です。プラグインとホストはそれぞれ別のプロセス上で動作し、ホストがmain.jsをWebWorkerとして呼び出すことでプラグインは起動します。以下はシンプルな例です。

  • main.js

addEventListener('message', onGetWave, false);

var sampleRate, currentRadian = 0, freqPerSample = false;

function onGetWave(e) {
	var request = e.data, response = {
		callback : request.callback
	};
	
	switch(request.action) {
		case 'init':
			sampleRate = request.samplerate;
			break;
		case 'midi':
			switch(request.type) {
				case "noteon":
					var freq = 440 * Math.pow(2, (request.note / 12) - 5) * 2 * Math.PI;
					freqPerSample = freq / sampleRate;
					break;
				case "noteoff":
				case "notealloff":
				default:
					freqPerSample = false;
					break;
			}
			break;
		case 'getbuffer':
			var buffer = new Float32Array(request.length);
			if(freqPerSample !== false) {
				for(var i = 0; i < request.length; i++) {
					currentRadian += freqPerSample;
					buffer[i] = Math.sin(currentRadian);
				}
			}
			response.leftbuffer = buffer;
			response.rightbuffer = buffer;
			break;
		default:
			return;
	}

	postMessage(response);
};

以下では上記サンプルのe.dataをリクエスト、responseをレスポンスと呼ぶことにします。 各リクエスト毎にレスポンスをホストに送信しなければなりません。

すべてのリクエストは以下の属性を共通で持ちます。

リクエスト属性 説明
@param {string} action リクエストの種類を表します。"init"、"set"、"midi"、"getbuffer"の4種類があります。
@param {string} callback ホスト側でコールバックで呼ばれる関数の名前です。レスポンスに属性として付加します。

リクエストとレスポンス

ここではタイプに応じたリクエスト、レスポンスの説明をします。

initリクエスト

プラグインをホストが起動して最初に呼ばれます。

リクエスト属性 説明
@param {number} samplerate サンプリング周波数[Hz]
レスポンス属性 説明
@param {Object} controller パラメータとGUIコントローラを設定します。

setリクエスト

プラグイン側で設定したパラメータにGUIコントローラなどから変更が加えられたときに呼ばれます。パラメータの設定方法については後述のGUIコントローラの説明で行います。

リクエスト属性 説明
@param {string} id 変更されたパラメータのid
@param {string|number} value 変更されたパラメータの値

ホスト側に送信するレスポンス属性はありません。

midiリクエスト

midiイベントを受信します。

リクエスト属性 説明
@param {string} type MIDIイベントのタイプです。"noteon"、"noteoff"、"notealloff"の種類があります。
@param {number} note "noteon"、"noteoff"の場合のMIDIノート番号です。
@param {number} verocity "noteon"の場合のベロシティです。範囲は[0, 1]です。

ホスト側に送信するレスポンス属性はありません。

getbufferリクエスト

波形バッファーを要求します。

リクエスト属性 説明
@param {number} length 要求するバッファーのサイズ
レスポンス属性 説明
@param {Float32Array} rightbuffer 右サンプルバッファーです。
@param {Float32Array} leftbuffer 左サンプルバッファーです。

GUIコントローラの設定

initリクエストに対するレスポンスのcontroller属性がGUI設定を行います。

controller.background

背景画像の設定を行います。

controller属性 説明
@param {string} image 背景画像の相対URLです
@param {string} height 背景画像の縦pixelです
@param {string} width 背景画像の幅pixelです

controller.controls

各GUIコントロールの情報を配列で設定します。コントロールには"knob"、"toggle"、"radio"の種類があり、各要素共通の属性は以下となります。

control共通属性 説明
@param {string} type コントロールのタイプを表します。"knob"、"toggle"、"radio"のうちどれかを指定します。
@param {string} id コントロールのidを指定します。setリクエストのid属性に対応します。
@param {string|number} value パラメータの初期値です。
@param {number} height コントロール画像の縦pixelです。
@param {number} width コントロール画像の幅pixelです。

knobコントロール

回転ノブです。ノブは真上を中心として270°回転します。

control属性 説明
@param {string} image コントロール画像の相対URLです。
@param {number} offsetx コントロール画像の背景画像左上を原点としたときの右方向オフセットpixelです。
@param {number} offsety コントロール画像の背景画像左上を原点としたときの下方向オフセットpixelです。
@param {number} min [optional]
最小値です。デフォルトは0です。
@param {number} max [optional]
最大値です。デフォルトは1です。
@param {number} step [optional]
ステップ値です。デフォルトは0.001です。
@param {boolean} label [optional]
数値のラベルを表示するフラグです。デフォルトはfalseです。
@param {number} labeloffsetx [optional]
label==trueのときに任意で右方向へのオフセット値を設定できます。
デフォルトは0でコントロールの回転中心を中心とした位置に配置されます。
@param {number} labeloffsety [optional]
label==trueのときに任意で下方向へのオフセット値を設定できます。
デフォルトは0でコントロールの回転中心を中心とした位置に配置されます。
@param {string} labelprefix [optional]
label==trueのときに任意でラベルの接頭文字を設定できます。
@param {string} labelpostfix [optional]
label==trueのときに任意でラベルの接尾文字を設定できます。

toggleコントロール

トグルボタンです。setリクエストのvalue属性にはbooleanが設定されます。

control属性 説明
@param {string} imageon オンのときの画像相対URLです。
@param {string} imageoff オフのときの画像の相対URLです。
@param {number} offsetx コントロール画像の背景画像左上を原点としたときの右方向オフセットpixelです。
@param {number} offsety コントロール画像の背景画像左上を原点としたときの下方向オフセットpixelです。

radioコントロール

ラジオボタンです。setリクエストのvalue属性には、下表のoffsets属性で設定したラジオボタン配列のインデックス値が設定されます。

control属性 説明
@param {string} imageon オンのときの画像相対URLです。
@param {string} imageoff オフのときの画像の相対URLです。
@param {Array} offsets @param {number} offsetx、@param {number} offsetyを要素としたオブジェクトの配列です。

デバッグ方法

WebWorker内ではconsole.logなどのデバッグ用のAPIが提供されていません。プラグイン内でのオブジェクトの中身を確認したい場合は、callback要素を持たないオブジェクトをpostMessage()してください。そうすればホスト側のJavascriptコンソールで確認することが出来ます。
